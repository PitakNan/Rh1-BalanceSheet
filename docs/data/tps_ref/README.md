# ข้อมูลอ้างอิง TPS (เขต 1 เท่านั้น)

สำเนา (กรองเฉพาะ 103 รพ. เขตสุขภาพที่ 1) จาก TPS (Total Performance Score, สำนักปลัด สธ.)
— ต้นฉบับเต็มอยู่ที่ `D:\Github\Rh1-TPS-V.3.1\public\data\` / https://rh1-tps-v-3-1.vercel.app

**ใช้เป็นข้อมูลอ้างอิง QA เท่านั้น** ไม่ใช่แหล่งข้อมูลหลักของ risk_drill.html เพราะ TPS อัพเดต
รายไตรมาส (ช้าเกินไปสำหรับ dashboard รายเดือน) — pipeline หลัก (`export_risk_link.py`) ยังคง
คำนวณจากงบทดลองทุกเดือนเหมือนเดิม ไฟล์นี้ใช้แค่เทียบว่าสูตรที่คำนวณเองใกล้เคียงแค่ไหน

## วิธีอัพเดตเมื่อมี TPS export ใหม่

1. เอาไฟล์ `financial_ratios.json` (และ `hospitals.json`/`periods.json` ถ้ามี) จาก TPS export
   ใหม่มาวางทับไฟล์ในโฟลเดอร์นี้ได้เลย — ไม่ต้อง filter เขต 1 ก่อนก็ได้ (ทั้งไฟล์ดิบ ~11MB
   หรือกรองแล้วเหลือ ~1MB ก็ใช้ได้ทั้งคู่ สคริปต์ตรวจสอบจะกรองเฉพาะ รพ. ที่มีอยู่จริงให้เอง)
2. รัน `python pipeline/validate_against_tps.py` จะได้สรุปว่าสูตรของเราใกล้เคียง TPS แค่ไหน
   (ควรรันทุกครั้งที่แก้สูตร 7 Plus Efficiency Score ในตัว pipeline หลักด้วย)
3. ดูรายชื่อ รพ. ที่ต่างเกินเกณฑ์ท้ายผลลัพธ์ไปตรวจสอบเพิ่มเติมถ้าจำเป็น
